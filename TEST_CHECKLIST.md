# 測試檢查清單

請在Godot中測試時，按照以下清單逐項驗證：

## 🔧 啟動檢查

- [ ] Godot專案能正常開啟
- [ ] 所有腳本沒有編譯錯誤（紅色標記）
- [ ] 資源重新匯入完成（可能需要等待30秒）

## 🎮 基本功能測試

### 遊戲啟動
- [ ] 遊戲可以正常啟動（F5）
- [ ] 主選單顯示正常
- [ ] 可以進入角色選擇

### 角色顯示
- [ ] 戰鬥開始後，左側顯示Player1精靈圖（藍色方塊）
- [ ] 右側顯示Player2精靈圖（綠色方塊）
- [ ] 精靈圖位置合理（不在螢幕外）
- [ ] 精靈圖大小合理（不會太小看不見）

### 動作執行
- [ ] 點選動作按鈕後，動作正常執行
- [ ] 執行動作時，使用者的精靈圖改變（變成紅色/紫色）
- [ ] 動作完成後，精靈圖恢復原狀（變回藍色/綠色）
- [ ] 戰鬥日誌正確記錄動作資訊

### 受擊效果
- [ ] 目標被攻擊時，精靈圖變色（變成橘色）
- [ ] 有閃光特效出現（白色閃光）
- [ ] 特效在約0.5秒後消失
- [ ] 受擊後精靈圖恢復原狀

### 戰鬥結束
- [ ] 戰鬥結束時顯示勝利訊息
- [ ] 勝利者的精靈圖可能改變（如果有victory.svg）
- [ ] 按任意鍵可以返回主選單

## 🐛 錯誤檢查

### 控制台訊息

**✅ 這些警告是正常的**：
```
AssetManager: 無法載入資源 'res://assets/sprites/hero/defend.png'，使用預設資源
AssetManager: 無法載入資源 'res://assets/audio/hero/attack.ogg'，使用預設資源
```

**❌ 這些錯誤不應該出現**：
```
AssetManager not found
CharacterVisualState not found
BattleVisualPlayer not found
Invalid get index 'asset_id'
Null instance
```

如果出現上述錯誤，請嘗試：
1. 場景 → 重新載入當前專案
2. 確認所有新腳本檔案存在
3. 手動開啟並儲存 Character.gd, Action.gd, StatusEffect.gd

## 🎨 視覺驗證

### 精靈圖檢查
- [ ] Hero待機：藍色方塊，中間有圓圈
- [ ] Hero攻擊：紅色方塊，中間有菱形
- [ ] Hero受擊：橘色方塊，中間有方形
- [ ] Elise待機：綠色方塊，中間有圓圈
- [ ] Elise攻擊：紫色方塊，中間有菱形
- [ ] Elise受擊：橘色方塊，中間有方形

### 動畫流程
1. 回合開始：雙方顯示待機精靈圖（藍色/綠色）
2. 選擇動作：精靈圖不變
3. 執行動作：使用者變攻擊精靈圖（紅色/紫色）
4. 命中目標：目標變受擊精靈圖（橘色）+ 白色閃光
5. 動作完成：雙方恢復待機精靈圖

### 特效檢查
- [ ] 每次攻擊都有特效
- [ ] 特效在目標位置顯示
- [ ] 特效會自動消失
- [ ] 場景樹中不會累積大量特效節點（按F4查看）

## 📊 效能測試

### 記憶體
- [ ] 長時間戰鬥後遊戲不會變慢
- [ ] 特效節點數量保持穩定（F4查看場景樹）
- [ ] 沒有記憶體洩漏警告

### 回應性
- [ ] 點選按鈕後立即有回應
- [ ] 動畫播放流暢，沒有卡頓
- [ ] 戰鬥日誌更新及時

## 🔍 進階測試（可選）

### 快取測試
1. 執行多個回合的戰鬥
2. 注意第2回合以後載入是否更快
3. 控制台查看"快取大小"相關訊息

### Fallback測試
1. 刪除其中一個精靈圖（如hero/attack.svg）
2. 重新執行遊戲
3. 確認使用fallback精靈圖（idle.svg）或預設精靈圖
4. 遊戲不會崩潰

### 邊界測試
1. 快速連續點選多個動作
2. 確認不會產生錯誤
3. 確認動畫按順序播放

## 🎯 最低通過標準

以下項目**必須**全部通過才算測試成功：

1. ✅ 遊戲能啟動且不崩潰
2. ✅ 角色精靈圖顯示在螢幕上
3. ✅ 執行動作時精靈圖會改變
4. ✅ 戰鬥可以正常進行到結束
5. ✅ 沒有嚴重的錯誤訊息（只有資源找不到的警告是可以接受的）

如果以上5項都通過，系統就算成功實作了！🎉

## 📞 需要協助？

1. **錯誤訊息**：完整的錯誤日誌（從控制台複製）
2. **測試步驟**：如何重現問題
3. **環境資訊**：Godot版本、作業系統
4. **螢幕截圖**：如果是視覺問題，提供截圖會很有幫助
