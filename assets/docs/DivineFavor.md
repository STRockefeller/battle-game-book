# 神恩系統

## 核心概念

**神恩**是對戰中的**臨時增益機制**，由參戰的神明提供。每場試煉都會有 5-6 位神明作為觀戰者，玩家可從中選擇一位作為該場試煉的庇佑者。被選中的神明會根據玩家的**具體行動**來評估玩家是否值得其加護。

### 關鍵設計特點

- **隱藏選擇**：玩家不知道對手選了哪位神明
- **隱藏任務**：玩家不知道對手的具體任務內容
- **漸進式揭露**：任務完成後，對手會看到你獲得的加護效果
- **推理對抗**：玩家需根據自己的神明選項初步推測對手的意圖，並在對手完成任務後動態調整策略
- **可選項**：不完成任務也是一種戰術選擇（可能迷惑對手或集中力量應對其他威脅）

---

## 神明體系

### 神明分佈

全遊戲共設 **12-15 位神明**，分為 5 大勢力：

| 勢力 | 象徵 | 神明數量 | 風格 |
|------|------|--------|------|
| **戰爭** | 紅色 | 3-4 | 攻擊、傷害、爆發 |
| **智慧** | 藍色 | 3-4 | 防守、迴避、控制 |
| **商業** | 金色 | 2-3 | 資源、效率、交易 |
| **生命** | 綠色 | 2-3 | 治療、保護、恢復 |
| **無所屬** | 紫色 | 2-3 | 平衡、詭計、特殊 |

### 神明選擇機制

**準備階段**：
1. 系統從 12-15 位神明中隨機挑選 **5 個不同勢力**的神明
   - 每個勢力最多 1 位（"無所屬"例外，可出現 2 個以上）
2. 如果選擇**預設角色**（故事模式中的可選角色），額外增加該**角色主保護神**（不重複）
   - 例如：埃莉絲選擇"自然之神"，不會與前 5 個重複
3. 玩家從可用神明中選擇 1 位作為該場試煉的庇佑者

**信息對稱**：雙方看到的可用神明池相同 → 可以根據對方的選擇時間推測其傾向

## 任務系統

### 任務結構

每位神明提供 **3 個難度選項**的任務，玩家在戰鬥開始時選擇：

```
神明的任務
├─ 🟢 簡單 (Basic)
│  └─ 要求較低，獎勵較小
├─ 🟡 標準 (Standard)  
│  └─ 平衡的要求和獎勵
└─ 🔴 困難 (Hard)
   └─ 高難度挑戰，豐厚獎勵
```

**選擇時機**：戰鬥開始前（與神明選擇分開的步驟）

**選擇可見性**：
- 玩家**能看到自己和對手的神明名稱**
- 玩家**不能看到對手的任務難度選擇**
- 玩家**不能看到對手的任務具體內容**

## 神明與任務庫

### 1. 戰神（攻擊導向）

#### 任務類型：進攻激進

**🟢 簡單：初試鋒芒**
- 要求：前 3 回合中至少進行 1 次攻擊行動
- 獎勵：攻擊力 +15% （持續整場戰鬥）

**🟡 標準：連貫進攻**
- 要求：前 5 回合中至少進行 3 次攻擊行動
- 獎勵：攻擊力 +30%，首次命中時必定先手 1 回合

**🔴 困難：不懈戰士**
- 要求：整場戰鬥中攻擊行動不少於 50% 的總行動數
- 獎勵：攻擊力 +50%，每次攻擊命中時額外恢復 5 HP

---

### 2. 智慧女神（防守導向）

#### 任務類型：防禦與迴避

**🟢 簡單：初試防禦**
- 要求：前 4 回合中至少進行 1 次防禦行動
- 獎勵：受傷害 -12%

**🟡 標準：堅壁清野**
- 要求：前 6 回合中至少進行 3 次防禦行動，期間不被命中超過 1 次
- 獎勵：受傷害 -25%，防禦時耐力消耗 -50%

**🔴 困難：不破之堡**
- 要求：10 回合內防禦成功（敵方未命中）10 次以上
- 獎勵：受傷害 -40%，每次成功防禦時回復 10 MP

---

### 3. 商業之神（資源導向）

#### 任務類型：效率與交易

**🟢 簡單：初嘗獲利**
- 要求：5 回合內使用 3 個不同行動
- 獎勵：耐力恢復速度 +20%

**🟡 標準：精打細算**
- 要求：整場戰鬥中行動多樣性達 70%（使用超過 70% 的可用行動）
- 獎勵：所有資源消耗 -20%（HP 治療除外），戰鬥結束額外獲得 1 點勝利點數

**🔴 困難：商業帝國**
- 要求：不浪費任何耐力（耐力點數始終在 80% 以上）且使用所有可用行動至少 1 次
- 獎勵：資源消耗 -40%，每回合自動恢復 15 耐力

---

### 4. 治癒女神（保護導向）

#### 任務類型：生命與恢復

**🟢 簡單：初次庇護**
- 要求：保持 HP 高於最大值 60% 持續 4 回合
- 獎勵：每回合自動恢復 5 HP

**🟡 標準：精心呵護**
- 要求：全場 HP 始終高於最大值 50%，且使用 2 次或以上恢復類行動
- 獎勵：每回合恢復 15 HP，受到致命傷害時自動保留 1 HP（1 次）

**🔴 困難：永恆守護**
- 要求：全場 HP 始終高於 70%，或在 HP 跌至危急時使用恢復行動 3 次以上
- 獎勵：每回合恢復 30 HP，免疫 1 次致命傷害

---

### 5. 詭計之神（特殊導向）

#### 任務類型：變化與欺瞞

**🟢 簡單：初露端倪**
- 要求：前 3 回合中改變姿態或使用特殊行動至少 1 次
- 獎勵：下一回合行動速度 +1（優先度提升）

**🟡 標準：虛實交錯**
- 要求：在對手預期的行動類型（攻擊/防守）上出其不意 3 次
  - （機制：如果上一回合使用攻擊，本回合改用防守，計數 +1；反之亦然）
- 獎勵：每次命中時有 30% 機率閃避對手的下一次攻擊

**🔴 困難：大師欺瞞**
- 要求：整場戰鬥中保持行動的不可預測性（5 回合內不重複同類行動超過 2 次）
- 獎勵：敵方對你的攻擊命中率 -30%，你的非攻擊行動獲得偽裝效果

---

### 6. 平衡之神（均衡導向）

#### 任務類型：中庸之道

**🟢 簡單：尋求平衡**
- 要求：行動類型相對均衡（攻擊、防守、特殊各至少 1 次）
- 獎勵：所有傷害 +10%，所有傷害承受 -10%

**🟡 標準：完美協調**
- 要求：攻擊、防守、特殊行動的使用次數差異不超過 1
- 獎勵：攻擊力 +20%，受傷害 -15%

**🔴 困難：天地平衡**
- 要求：HP 與耐力管理完美（HP 始終在 40-90% 之間，耐力始終在 30-90% 之間）
- 獎勵：所有數值 +25%（傷害、防禦、恢復、效率）

## 任務進度與顯示

### 戰鬥中的任務追蹤

**玩家視角**：
- 在戰鬥界面左上角顯示自己的任務進度（例如「進攻激進 3/5 回合」）
- **看不到對手的任務內容或進度**

**對手視角**：
- 對手同樣看不到你的任務內容或進度
- 但可以通過你的行動模式推測

### 任務完成時

**觸發時機**：任務完成的瞬間

**視覺反饋**：
1. 玩家端顯示完成提示（例如「戰神認可你的進攻！」）
2. 加護效果文字浮出（例如「攻擊力 +30%」）
3. **對手端會看到你獲得的加護效果名稱與數值**
   - 例如：「對手獲得【戰神的加護】：攻擊力 +30%」

**作用時機**：加護效果立即生效，影響後續計算

## 預設角色的主保護神

每位預設角色綁定一位**主保護神**，該神明在玩家選擇對應角色時作為第 6 個選項出現：

| 角色 | 主保護神 | 勢力 | 原因 |
|------|--------|------|------|
| **埃莉絲** | 自然之神 | 無所屬 | 林地精靈，與自然共生 |
| **英雄** | 戰神 | 戰爭 | 勇敢的戰士 |
| **賽拉** | 商業之神 | 商業 | 煉金術士，追求效率 |
| **米娜** | 詭計之神 | 無所屬 | 夢行者，虛實莫辨 |
| **西魯斯** | 平衡之神 | 無所屬 | 皇族指揮官，兼備全能 |
| **拉格** | 治癒女神 | 生命 | 獄民出身，懂得珍視生命 |

**設計意義**：
- 新手更容易理解預設角色的戰術風格
- 故事模式中可藉此強化角色背景
- 對戰中成為玩家的"安全選項"

## 神恩系統與其他機制的互動

### 與被動特質的協同

**推理機制**：
- 如果對手選了"傷害加成"特質 → 可能傾向選戰神或詭計之神
- 如果對手選了"資源優化"特質 → 可能傾向選商業之神
- 如果對手選了"防守加成"特質 → 可能傾向選智慧女神

**對抗策略**：
- 瞭解特質後推測神明選擇，選擇相剋的神明
- 例如：對手選了傷害特質 + 推測戰神 → 你選智慧女神並選困難防守任務

### 與行動配置的協同

**行動與任務的匹配度**：
- 某些行動組合能更容易完成特定任務
- 例如：配置了多個攻擊招式 → 完成戰神的進攻任務更容易
- 例如：配置了防守和恢復招式 → 完成治癒女神的保護任務更容易

## 戰鬥流程中的神恩

### 準備階段集成

```
對戰準備流程：
1. 角色選擇
2. 被動特質選擇
3. 行動配置
4. 神恩選擇
   ├─ 選擇神明（可用神明池相同，雙方可相互推測）
   └─ 選擇任務難度（對方看不到）
5. 戰鬥開始
```

### 戰鬥迴圈中的神恩

```
每回合流程：
├─ 回合開始
├─ 行動選擇
├─ 結果應用
│  ├─ 傷害計算（應用所有加護效果）
│  └─ 任務進度更新
├─ 任務完成檢測
│  └─ 若完成 → 通知對手並應用加護
└─ 回合結束
```

## 戰術深度與推理

### 玩家的推理層次

**第 1 層：神明推測**
- 看到對手選擇的神明名稱
- 推測對手可能的任務類型
- 提前準備應對策略

**第 2 層：難度推測**
- 根據對手的前幾回合行動推測任務難度
- 例如：對手頻繁攻擊 → 可能是戰神的簡單或標準難度
- 例如：對手謹慎防守 → 可能是智慧女神的困難難度

**第 3 層：動態應對**
- 看到對手完成的加護效果後，重新評估策略
- 例如：對手獲得「攻擊力 +50%」→ 改為專注防守和恢復
- 例如：對手獲得「受傷害 -40%」→ 改為使用狀態效果和特殊行動

### 玩家的戰術選擇

**選項 1：主動完成任務**
- 優點：獲得強大加護，影響對手策略
- 缺點：行動會被看透，可能被預測
- 時機：對自己的特質和行動配置有信心時

**選項 2：隱瞞意圖**
- 優點：迷惑對手，讓其誤判
- 缺點：失去加護優勢
- 時機：對手的加護已經很強，不如藏著力量反擊

**選項 3：假動作**
- 優點：讓對手誤以為你在做某個任務
- 缺點：如果對手看破，會更加警惕
- 時機：對手進度領先，需要製造混亂時

## 實現參考

### 資源需求

- **12-15 位神明的設計**（角色設計、背景故事、視覺資源）
- **每位神明 3 個難度任務**（共 36-45 個任務條目）
- **任務進度追蹤系統**（數據結構、判定邏輯）
- **UI 組件**（神明選擇界面、任務進度條、加護效果顯示）
- **視覺與音效反饋**（完成動畫、通知提示音）

### 資料結構示例

```gdscript
# 神明資料
class_name Deity
var id: String              # "god_of_war"
var name: String            # "戰神"
var faction: String         # "war" / "wisdom" / "commerce" / "life" / "none"
var description: String     # 背景故事
var sprite: Texture         # 視覺資源
var tasks: Array[Task]      # 3 個難度任務

# 任務資料
class_name Task
var difficulty: int         # 0=簡單, 1=標準, 2=困難
var requirement: String     # 任務要求（文字描述）
var reward: Dictionary      # { "damage": 0.3, "healing": 0.2, ... }
var progress: int           # 當前進度
var completed: bool         # 是否完成

# 神恩效果
class_name DivineBlessing
var deity_id: String
var task_difficulty: int
var effects: Dictionary     # { "damage_bonus": 0.3, "defense_reduction": 0.25, ... }
var active: bool
```
