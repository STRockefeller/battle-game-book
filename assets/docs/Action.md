# Action

### 動作（Action）設計文件

---

#### 核心參數

每個動作都將是一個獨立的類別，包含以下核心參數：

* **id**：動作的唯一標識符（例如："QuickSlash", "Fireball"）。
* **name**：動作的名稱。
* **description**：動作的詳細說明。
* **cost**：發動此動作所需的**耐力**。
* **cast_time**：**詠唱時間**，單位為回合。詠唱期間，角色無法進行其他行動，且容易受到攻擊。
* **cooldown**：**冷卻時間**，單位為回合。使用後，動作將在一段時間內無法再次使用。
* **tags**：動作的標籤，用於觸發特效和機制互動（例如：`Slash`, `Fire`, `AreaOfEffect`）。

#### 戰鬥數值

這些參數將決定動作的實際戰鬥效果：

* **damage_type**：傷害類型（例如：`Physical`, `Magic`）。
* **damage_multiplier**：傷害倍率。最終傷害由角色的攻擊力乘以這個倍率得出。
* **accuracy_modifier**：命中修正值，加到角色的基礎命中率上。
* **critical_modifier**：暴擊修正值，加到角色的基礎暴擊率上。

#### 效果與交互

* **effects**：一個列表，包含此動作所能施加的**狀態效果**。
    * **觸發時機**：你可以設定效果是**命中後生效**，還是**使用時立即生效**。
* **stance_change**：動作會改變自己或敵人的姿態（例如：造成倒地、使目標滯空）。

#### 遊戲機制

* **詠唱時間 (Cast Time)**：
    * 如果一個動作的詠唱時間大於 0，角色將在當前回合進入「詠唱」姿態。
    * 在詠唱期間，角色無法進行任何其他行動，但可能允許選擇「中斷詠唱」。
    * 若詠唱未被中斷，則在詠唱時間結束後，動作將會自動發動。
    * 這能讓高傷害或強控場的動作有明確的風險。
* **冷卻時間 (Cooldown)**：
    * 動作發動後，其冷卻時間會啟動。
    * 冷卻時間歸零前，該動作將從角色的動作清單中隱藏或變灰。
    * 這能避免玩家無腦重複使用最強動作，鼓勵動作輪換。
* **標籤交互 (Tag Interactions)**：
    * 在計算傷害時，你可以檢查動作標籤和目標的狀態。例如，一個帶有 `Water` 標籤的動作攻擊處於 `Burning` 狀態的目標時，可以獲得額外傷害，並移除 `Burning` 狀態。
    * 這增加了戰鬥的策略深度，鼓勵玩家利用不同的動作組合。

---

### 補充

* **詠唱時間**：在回合制中，你可以讓角色在詠唱期間進入一個名為「詠唱」的特殊**姿態**。這將完美契合我們之前討論的機制，使角色在詠唱期間只能被動防禦，無法攻擊。
* **冷卻時間**：這會是動作本身的一個屬性，由角色來管理。每回合結束時，角色可以檢查所有動作的冷卻時間並減一。
* **強度指標**：將詠唱和冷卻時間加入強度指標後，公式會更複雜，但也會更精確。
    * **詠唱時間**：每多一個詠唱回合，動作的強度值就會降低，因為這是一個巨大的負面效果。
    * **冷卻時間**：冷卻時間越長，動作的強度值就越高。
