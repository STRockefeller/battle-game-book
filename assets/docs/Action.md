# Action

### 動作（Action）設計文件

---

#### 核心參數

每個動作都將是一個獨立的類別，包含以下核心參數：

* **id**：動作的唯一標識符（例如："QuickSlash", "Fireball"）。
* **name**：動作的名稱。
* **description**：動作的詳細說明。
* **cost**：發動此動作所需的**耐力**。
* **cast_time**：**詠唱時間**，單位為回合。詠唱期間，角色無法進行其他行動，且容易受到攻擊。
* **cooldown**：**冷卻時間**，單位為回合。使用後，動作將在一段時間內無法再次使用。
* **tags**：動作的標籤，用於觸發特效和機制互動（例如：`Slash`, `Fire`, `AreaOfEffect`）。

#### 戰鬥數值

這些參數將決定動作的實際戰鬥效果：

* **damage_type**：傷害類型（例如：`Physical`, `Magic`）。
* **damage_multiplier**：傷害倍率。最終傷害由角色的攻擊力乘以這個倍率得出。
* **accuracy_modifier**：命中修正值，加到角色的基礎命中率上。
* **critical_modifier**：暴擊修正值，加到角色的基礎暴擊率上。

#### 效果與交互

* **effects**：一個列表，包含此動作所能施加的**狀態效果**。
    * **觸發時機**：你可以設定效果是**命中後生效**，還是**使用時立即生效**。
* **stance_change**：動作會改變自己或敵人的姿態（例如：造成倒地、使目標滯空）。

#### 遊戲機制

* **詠唱時間 (Cast Time)**：
    * 若動作詠唱時間大於 0，角色將進入「詠唱」姿態。
    * 詠唱期間角色無法進行其他行動。
    * 詠唱完成後動作自動發動。
* **冷卻時間 (Cooldown)**：
    * 動作發動後啟動冷卻時間。
    * 冷卻時間歸零前，該動作將從角色動作清單中隱藏。
* **標籤交互 (Tag Interactions)**：
    * 計算傷害時可檢查動作標籤和目標狀態。例如，帶有 `Water` 標籤的動作攻擊 `Burning` 目標時可獲得額外傷害，並移除 `Burning` 狀態。
