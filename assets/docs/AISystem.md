# AI 系統設計

## 核心概念

AI 系統提供對手角色自動決策機制。遊戲支援多種 AI 風格，允許玩家選擇對手難度或行為模式。

## AI 類型

### 1. 隨機型 AI（Random）

**特性**:
- 每回合完全隨機選擇可用動作
- 無策略邏輯，作為基準測試

**適用場景**:
- 教學模式
- 調試和測試

**遊戲體驗**:
- 難度最低
- 行為不可預測但無威脅

---

### 2. 攻擊型 AI（Aggressive）

**特性**:
- 優先選擇高傷害動作
- 傾向於持續進攻
- 血量低時仍然進攻

**決策邏輯**:
```
優先度 = 基礎傷害 + 傷害倍率加成 + 命中率加成 - 風險懲罰
```

**適用場景**:
- 敵對 Boss
- 較高難度設置

**遊戲體驗**:
- 高風險、高傷害
- 明確的威脅感
- 玩家需要快速決策

---

### 3. 防守型 AI（Defensive）

**特性**:
- 優先使用防禦和治療動作
- 血量低時立即防守或恢復
- 傾向於低消耗動作
- 持久戰型

**決策邏輯**:
```
血量 < 30% → 優先防禦/恢復
耐力 < 30% → 使用休息
其他情況 → 低消耗攻擊
```

**適用場景**:
- 防守方敵人
- 中等難度對手

**遊戲體驗**:
- 戰鬥延長
- 對手難以被快速擊敗
- 玩家需要持久力

---

### 4. 平衡型 AI（Balanced）

**特性**:
- 攻防並重
- 根據自身狀態動態調整策略
- 適度使用各類動作
- 綜合難度

**決策邏輯**:
```
if 自身血量低 then 防禦/恢復
else if 對手血量高 then 高傷害攻擊
else 高命中攻擊
```

**適用場景**:
- 標準對手
- 推薦難度

**遊戲體驗**:
- 平衡挑戰
- 易於上手
- 類似玩家策略

## AI 參數

### 通用參數

| 參數 | 範圍 | 說明 |
| :--- | :--- | :--- |
| **攻擊性** (aggression) | 0.0-1.0 | 傾向高傷害動作的程度 |
| **謹慎度** (caution) | 0.0-1.0 | 優先防禦/避免風險的程度 |
| **技能使用率** (skill_usage) | 0.0-1.0 | 使用特殊技能的頻率 |

### 高級參數

| 參數 | 說明 |
| :--- | :--- |
| **血量閾值** (hp_threshold_low) | 觸發防禦策略的血量百分比 |
| **魔量閾值** (mp_threshold_low) | 觸發恢復或保留資源的魔量百分比 |
| **優先高傷害** (prefer_high_damage) | 是否偏好單次大傷害而非穩定輸出 |

## 決策流程

```
┌──────────────────────────────────┐
│  AI 接收可用動作列表              │
├──────────────────────────────────┤
│  1. 評估當前戰況                 │
│     - 自身血量、魔量、耐力       │
│     - 對手血量、狀態             │
│                                  │
│  2. 根據 AI 類型評分每個動作     │
│     - 計算優先度分數             │
│     - 加入隨機性避免預測         │
│                                  │
│  3. 選擇最高分動作               │
│     - 或加權隨機選擇前 N 個      │
│                                  │
│  4. 返回選擇給對戰管理器          │
└──────────────────────────────────┘
```

## 設計特性

### 可預測但不死板

AI 行為基於明確的邏輯規則，但加入隨機性：
- 同一情況下 AI 不會做出完全相同的選擇
- 玩家可以學習 AI 傾向但無法完全預測

### 參數可調

不同難度或敵人可配置不同參數：
```
簡單: 攻擊性=0.3, 謹慎度=0.7, 低血量閾值=0.1
中等: 攻擊性=0.5, 謹慎度=0.5, 低血量閾值=0.3
困難: 攻擊性=0.8, 謹慎度=0.2, 低血量閾值=0.5
```

### 可視化決策依據

玩家可在戰鬥中看到 AI 選擇的動作類型：
- 選擇防禦 → 預示 AI 血量低
- 選擇高傷害技能 → 預示 AI 進攻意圖
- 選擇治療 → 預示 AI 保留資源

## 平衡考量

### 難度設計

AI 難度應通過以下因素調整，而非單純修改傷害：
1. **動作選擇智慧** - 更智慧的決策
2. **反應時機** - 更好的時間控制
3. **資源管理** - 更高效的魔量/耐力使用
4. **隊型策略** - 多角色時的配合

### 公平性

確保 AI 與玩家遵循相同規則：
- ✅ 受同樣的冷卻限制
- ✅ 受同樣的資源限制（MP/STA）
- ✅ 受同樣的效果影響
- ✅ 不應有特殊優待或隱藏加成

## 擴展可能

### 短期
- 新增專為特定 Boss 設計的 AI
- 根據遊戲進度調整 AI 難度
- 記錄玩家行為以動態調整 AI 策略

### 中期
- 實現 AI 學習系統
- 多 AI 配合的隊伍策略
- AI 行為樹編輯器

### 長期
- 神經網路 AI（可選）
- 玩家創建自訂 AI 的工具
- AI 聯機對戰
